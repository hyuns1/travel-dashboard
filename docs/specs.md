# 여행 플래너 대시보드 — 페이지 개요 (specs)

## 앱 한 줄 소개
도시 이름 하나를 입력하면, 그 도시의 날씨·환율·국가정보·지도를
한 곳에서 보여주는 여행 준비용 웹 대시보드.

## 공통 동작
- 메인에서 도시를 한 번 검색하면 st.session_state에 저장됨
- 나머지 페이지는 저장된 도시를 가져와 자동으로 같은 도시를 표시
- 모든 API는 키 불필요 / @st.cache_data로 중복 요청 방지 / try-except로 에러 처리

## 1. 메인 (app.py) — 담당: B
- 입력: 사용자가 입력한 도시 이름
- 출력: 도시를 저장하고 각 페이지로 안내 (통합의 중심)
- 사용: utils/geocode.py

## 2. 날씨 페이지 (pages/1_날씨.py) — 담당: A
- 입력: 저장된 도시(위경도)
- 출력: 현재 날씨 카드 + 7일 예보 차트
- API: Open-Meteo

## 3. 환율 페이지 (pages/2_환율.py) — 담당: 이현성
- 입력: 금액, 기준 통화, 대상 통화
- 출력: 환산 금액 (추세 차트는 보너스)
- API: Frankfurter

## 4. 국가정보 페이지 (pages/3_국가정보.py) — 담당: 이현성
- 입력: 저장된 도시(→ 국가)
- 출력: 국기, 수도, 인구, 통화, 언어 카드
- API: REST Countries

## 5. 지도 페이지 (pages/4_지도.py) — 담당: B
- 입력: 저장된 도시(위경도)
- 출력: folium 지도에 도시 위치 핀
- 사용: geocode 재사용 + OpenStreetMap

## 공용 부품
- utils/geocode.py (도시명 → 위경도): A가 먼저 만들어 공용. 날씨·지도·국가정보가 사용.

## 역할 요약
- A: 날씨 페이지 + 공용 geocode 함수
- B: 메인 + 통합(session_state) + 지도 + 전체 UI
- 이현성(hyuns1): 환율·국가정보 페이지 + 배포·발표·문서
